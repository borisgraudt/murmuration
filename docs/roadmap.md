--- ### Шаг 2.1: AI-Routing Scoring

**Файлы:**
- `core/src/p2p/routing.rs` - добавить scoring
- `core/src/p2p/peer.rs` - добавить метрики

**Код:**
```rust
pub struct PeerMetrics {
  pub latency: Duration,
  pub uptime: Duration,
  pub packet_loss: f32,
  pub last_seen: Instant,
}

pub fn calculate_score(metrics: &PeerMetrics) -> f64 {
  0.7 * metrics.uptime.as_secs() as f64 / 3600.0 +  // uptime в часах
  0.3 * (1.0 - metrics.latency.as_millis() as f64 / 1000.0)  // latency нормализованная
}
```

**Время:** 6-8 часов

--- ## Критические риски и решения

### Риск 1: Не хватит времени
**Решение:** Сфокусироваться на core + CLI + базовый web. Сайты можно упростить.

### Риск 2: Сложность интеграции Rust ↔ Python
**Решение:** Использовать JSON-RPC через TCP сокет. Просто и надежно.

### Риск 3: AI-routing не работает
**Решение:** На MVP использовать rule-based routing с метриками. ML можно добавить позже.
